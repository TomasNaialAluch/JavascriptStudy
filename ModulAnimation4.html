setTimeout - Ejecución Diferida

console.log("1. Inicio");
setTimeout(() => {
console.log("3. Después de 2 segundos");
}, 2000);
console.log("2. Fin");
1. Inicio
setTimeout() programado
2. Fin
3. Después de 2 segundos
JavaScript no espera - continúa ejecutando

Explicación de setTimeout

setTimeout programa una función para ejecutarse después de un tiempo específico:

  • No bloquea: JavaScript continúa ejecutando el resto del código
  • Asíncrono: La función se ejecuta cuando termina el tiempo
  • Callback: La función es un callback que se ejecuta después
  • Orden: "Fin" se ejecuta antes que el callback

Promises - Manejo de Asincronía

let promesa = new Promise((resolve, reject) => {
setTimeout(() => {
resolve("¡Éxito!");
}, 1000);
});
promesa.then(resultado => {
console.log(resultado);
});
new Promise()
Pending
Resolved: "¡Éxito!"
.then() ejecutado
Promise resuelta → .then() se ejecuta

Explicación de Promises

Las Promises representan una operación asíncrona que puede completarse en el futuro:

  • Estados: Pending → Resolved/Rejected
  • resolve(): Marca la promise como exitosa
  • .then(): Se ejecuta cuando la promise se resuelve
  • Ventaja: Evita el "callback hell"

Async/Await - Sintaxis Moderna

async function obtenerDatos() {
console.log("1. Iniciando...");
let resultado = await fetch('/api/datos');
console.log("2. Datos obtenidos");
return resultado;
}
1. Iniciando...
await fetch() - Esperando
2. Datos obtenidos
return resultado
async/await hace que el código asíncrono se vea síncrono

Explicación de Async/Await

async/await es una forma más limpia de trabajar con Promises:

  • async: Marca la función como asíncrona
  • await: Espera a que la Promise se resuelva
  • Ventaja: Código más legible y fácil de debuggear
  • Uso: Ideal para operaciones secuenciales
