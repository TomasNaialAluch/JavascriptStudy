Variables y Tipos de Datos

Las variables son contenedores para almacenar datos. En JavaScript, puedes declarar variables usando var, let o const.

🎯 Puntos Clave

3 formas de declarar variables
7 tipos de datos principales
Scope y hoisting importantes
Type coercion y comparaciones
Closures y lexical scope

Teoría

Declaración de Variables

var Declaración tradicional, con scope de función
let Declaración moderna, con scope de bloque
const Constante que no puede ser reasignada

Tipos de Datos

String Texto entre comillas
Number Números enteros o decimales
Boolean true o false
Object Colección de propiedades
Array Lista ordenada de elementos

Ejemplo Práctico

Declaración Básica de Variables 
// Declaración de variables
var nombre = "Juan";           // String
let edad = 25;                 // Number
const PI = 3.14159;           // Number (constante)

// Tipos de datos básicos
let texto = "Hola mundo";      // String
let numero = 42;               // Number
let booleano = true;           // Boolean

console.log("Nombre:", nombre);
console.log("Edad:", edad);
console.log("PI:", PI);
Objetos y Arrays 
// Objeto con propiedades
let usuario = {
    nombre: "María García",
    edad: 28,
    activo: true,
    hobbies: ["programar", "leer", "viajar"]
};

// Acceso a propiedades
console.log("Nombre:", usuario.nombre);
console.log("Hobbies:", usuario.hobbies);
console.log("Primer hobby:", usuario.hobbies[0]);

Operadores

Los operadores permiten realizar operaciones con valores. JavaScript incluye operadores aritméticos, de comparación y lógicos para manipular datos.

🎯 Puntos Clave

6 operadores aritméticos
8 operadores de comparación
3 operadores lógicos
Precedencia de operadores
Type coercion importante

Teoría

Operadores Aritméticos

+ Suma
- Resta
* Multiplicación
/ División
% Módulo (resto)
** Exponenciación

Operadores de Comparación

== Igualdad débil
=== Igualdad estricta
!= Desigualdad débil
!== Desigualdad estricta
> < >= <= Comparaciones numéricas

Operadores Lógicos

&& AND lógico
|| OR lógico
! NOT lógico

Ejemplo Práctico

Operadores Básicos 
// Operadores aritméticos
let a = 10;
let b = 3;

console.log('Suma:', a + b);        // 13
console.log('Resta:', a - b);       // 7
console.log('Multiplicación:', a * b); // 30
console.log('División:', a / b);    // 3.33...
console.log('Módulo:', a % b);      // 1
console.log('Exponenciación:', a ** b); // 1000

// Operadores de comparación
let x = 5;
let y = "5";

console.log('Igualdad débil (==):', x == y);   // true
console.log('Igualdad estricta (===):', x === y); // false
console.log('Mayor que:', x > 3);    // true
console.log('Menor que:', x < 10);   // true

// Operadores lógicos
let esAdulto = true;
let tieneLicencia = false;

console.log('AND (&&):', esAdulto && tieneLicencia);  // false
console.log('OR (||):', esAdulto || tieneLicencia);   // true
console.log('NOT (!):', !esAdulto);                   // false
Precedencia y Type Coercion 
// Precedencia de operadores
console.log('Sin paréntesis:', 2 + 3 * 4);     // 14 (multiplicación primero)
console.log('Con paréntesis:', (2 + 3) * 4);   // 20

// Type coercion (conversión automática de tipos)
console.log('String + Number:', "5" + 3);      // "53" (concatenación)
console.log('Number + String:', 5 + "3");      // "53" (concatenación)
console.log('String - Number:', "5" - 3);      // 2 (resta convierte a número)
console.log('String * Number:', "5" * 3);      // 15 (multiplicación convierte a número)

// Comparaciones con type coercion
console.log('"5" == 5:', "5" == 5);           // true (conversión)
console.log('"5" === 5:', "5" === 5);         // false (sin conversión)
console.log('null == undefined:', null == undefined); // true
console.log('null === undefined:', null === undefined); // false

// Operadores de asignación
let numero = 10;
numero += 5;  // numero = numero + 5
console.log('+= 5:', numero);  // 15

numero *= 2;  // numero = numero * 2
console.log('*= 2:', numero);  // 30

// Operadores de incremento/decremento
let contador = 5;
console.log('Pre-incremento:', ++contador);  // 6
console.log('Post-incremento:', contador++); // 6 (pero contador ahora es 7)
console.log('Valor final:', contador);       // 7

  • Los operadores aritméticos realizan operaciones matemáticas básicas
  • Los operadores de comparación evalúan relaciones entre valores
  • Los operadores lógicos combinan expresiones booleanas
  • La precedencia determina el orden de evaluación
  • Type coercion puede causar comportamientos inesperados
Lección 3

Estructuras de Control

Las estructuras de control permiten que tu código tome decisiones y repita operaciones basándose en condiciones. Son fundamentales para crear lógica en JavaScript.

🎯 Puntos Clave

3 tipos de condicionales
4 tipos de bucles
Operador ternario
Break y continue
Iteración de arrays y objetos

Teoría

Condicionales

if/else Ejecuta código basado en condición
switch Evalúa múltiples casos con case
? : Operador ternario (una línea)

Bucles (Loops)

for Bucle con contador (índices)
while Mientras se cumple condición
for...of Iterar sobre VALORES de arrays
for...in Iterar sobre PROPIEDADES de objetos

Control de Flujo

break Salir del bucle
continue Saltar iteración
return Salir de función

Ejemplo Práctico

Condicionales y Bucles Básicos 
// Condicionales
let temperatura = 25;

if (temperatura > 30) {
    console.log("Hace mucho calor");
} else if (temperatura > 20) {
    console.log("Temperatura agradable");
} else if (temperatura > 10) {
    console.log("Hace fresco");
} else {
    console.log("Hace frío");
}

// Operador ternario
let mensaje = temperatura > 25 ? "Calor" : "Fresco";
console.log("Mensaje:", mensaje);

// Switch
let dia = "lunes";

switch (dia) {
    case "lunes":
        console.log("Inicio de semana");
        break;
    case "viernes":
        console.log("¡Fin de semana!");
        break;
    case "sábado":
    case "domingo":
        console.log("Día de descanso");
        break;
    default:
        console.log("Día laboral");
}

// Bucles básicos
console.log("--- For Loop ---");
for (let i = 0; i < 5; i++) {
    console.log(`Iteración ${i}`);
}

console.log("--- While Loop ---");
let contador = 0;
while (contador < 3) {
    console.log(`Contador: ${contador}`);
    contador++;
}
Iteración de Arrays y Objetos 
// Arrays
let frutas = ["manzana", "banana", "naranja", "uva"];

console.log("--- For...of (valores) ---");
for (let fruta of frutas) {
    console.log(fruta);
}

console.log("--- For tradicional con índices ---");
for (let i = 0; i < frutas.length; i++) {
    console.log(`${i}: ${frutas[i]}`);
}

// Objetos
let persona = { 
    nombre: "Juan", 
    edad: 30, 
    ciudad: "Madrid",
    profesion: "Desarrollador"
};

console.log("--- For...in (propiedades) ---");
for (let propiedad in persona) {
    console.log(`${propiedad}: ${persona[propiedad]}`);
}

// Break y Continue
console.log("--- Break y Continue ---");
for (let i = 0; i < 10; i++) {
    if (i === 3) {
        console.log("Saltando 3...");
        continue; // Salta esta iteración
    }
    if (i === 7) {
        console.log("Llegamos a 7, terminando...");
        break; // Sale del bucle
    }
    console.log(`Número: ${i}`);
}

// Bucle anidado
console.log("--- Bucle Anidado ---");
for (let i = 1; i <= 3; i++) {
    for (let j = 1; j <= 2; j++) {
        console.log(`i=${i}, j=${j}`);
    }
}

📝 Resumen de la Lección

  • Los condicionales permiten tomar decisiones en el código
  • Los bucles repiten operaciones de manera eficiente
  • El operador ternario simplifica condicionales simples
  • Break y continue controlan el flujo de los bucles
  • for...of es ideal para arrays, for...in para objetos

Funciones

Las funciones son bloques de código reutilizables que realizan una tarea específica. Son fundamentales en JavaScript para organizar y reutilizar código.

🎯 Puntos Clave

3 tipos de funciones
Parámetros y argumentos
Return y scope
Arrow functions
Closures importantes

Teoría

Tipos de Funciones

  • Función declarada: function nombre() {}
  • Función expresada: let nombre = function() {}
  • Arrow function: let nombre = () => {}

Conceptos Importantes

  • Parámetros: Valores que recibe la función
  • Return: Valor que devuelve la función
  • Scope: Alcance de las variables
  • Hoisting: Elevación de declaraciones

Ejemplo Práctico

Declaración de Funciones 
// Función declarada
function saludar(nombre) {
    return `Hola, ${nombre}!`;
}

// Función expresada
let sumar = function(a, b) {
    return a + b;
};

// Arrow function (ES6)
let multiplicar = (a, b) => a * b;

// Función con parámetros por defecto
function crearUsuario(nombre, edad = 18, activo = true) {
    return {
        nombre,
        edad,
        activo
    };
}

// Scope y Hoisting
function ejemplo() {
    let local = "Soy local";
    var global = "Soy global";
}

// Hoisting con var
console.log(x); // undefined (no error)
var x = 5;

// No hoisting con let/const
// console.log(y); // ReferenceError
let y = 10;

// Uso de funciones
console.log(saludar("María"));           // "Hola, María!"
console.log(sumar(5, 3));                // 8
console.log(multiplicar(4, 6));          // 24
console.log(crearUsuario("Juan", 25));   // { nombre: "Juan", edad: 25, activo: true }

Arrays y Objetos

Los arrays y objetos son estructuras de datos fundamentales en JavaScript para organizar y manipular información.

Teoría

Arrays

  • Métodos básicos: push(), pop(), shift(), unshift()
  • Métodos de transformación: map(), filter(), reduce()
  • Búsqueda: indexOf(), includes(), find()

Objetos

  • Propiedades: Acceso con punto o corchetes
  • Métodos: Funciones dentro de objetos
  • Destructuring: Extraer propiedades

Ejemplo Práctico

Arrays y Objetos 
// Arrays
let numeros = [1, 2, 3, 4, 5];

// Métodos de array
numeros.push(6);           // [1, 2, 3, 4, 5, 6]
numeros.pop();             // [1, 2, 3, 4, 5]
numeros.unshift(0);        // [0, 1, 2, 3, 4, 5]
numeros.shift();           // [1, 2, 3, 4, 5]

// Métodos de transformación
let dobles = numeros.map(n => n * 2);        // [2, 4, 6, 8, 10]
let pares = numeros.filter(n => n % 2 === 0); // [2, 4]
let suma = numeros.reduce((acc, n) => acc + n, 0); // 15

// Búsqueda
let indice = numeros.indexOf(3);     // 2
let existe = numeros.includes(3);    // true
let encontrado = numeros.find(n => n > 3); // 4

// Objetos
let persona = {
    nombre: "Ana",
    edad: 25,
    saludar: function() {
        return `Hola, soy ${this.nombre}`;
    }
};

// Acceso a propiedades
console.log(persona.nombre);        // "Ana"
console.log(persona["edad"]);       // 25
console.log(persona.saludar());     // "Hola, soy Ana"

// Modificación
persona.ciudad = "Madrid";
persona.edad = 26;

// Destructuring
let { nombre, edad } = persona;
console.log(nombre, edad); // "Ana" 26

Ejercicio 1: Calculadora Básica

Principiante

Crea una calculadora que realice las operaciones básicas: suma, resta, multiplicación y división.

Requisitos:

  • Crear una clase Calculadora
  • Implementar métodos para cada operación
  • Manejar errores (división por cero)
  • Mantener historial de operaciones
// Solución del Ejercicio 1: Calculadora Básica
class Calculadora {
    constructor() {
        this.historial = [];
    }
    
    sumar(a, b) {
        let resultado = a + b;
        this.agregarAlHistorial(`${a} + ${b} = ${resultado}`);
        return resultado;
    }
    
    restar(a, b) {
        let resultado = a - b;
        this.agregarAlHistorial(`${a} - ${b} = ${resultado}`);
        return resultado;
    }
    
    multiplicar(a, b) {
        let resultado = a * b;
        this.agregarAlHistorial(`${a} * ${b} = ${resultado}`);
        return resultado;
    }
    
    dividir(a, b) {
        if (b === 0) {
            throw new Error('No se puede dividir por cero');
        }
        let resultado = a / b;
        this.agregarAlHistorial(`${a} / ${b} = ${resultado}`);
        return resultado;
    }
    
    agregarAlHistorial(operacion) {
        this.historial.push(operacion);
    }
    
    obtenerHistorial() {
        return this.historial;
    }
}

// Uso de la calculadora
let calc = new Calculadora();
console.log(calc.sumar(5, 3));        // 8
console.log(calc.multiplicar(4, 6));  // 24
console.log(calc.obtenerHistorial());

Ejercicio 2: Gestor de Tareas

Intermedio

Crea un sistema de gestión de tareas con funcionalidades CRUD y persistencia local.

Requisitos:

  • Agregar, editar y eliminar tareas
  • Marcar tareas como completadas
  • Filtrar por estado
  • Guardar en localStorage
// Solución del Ejercicio 2: Gestor de Tareas
class GestorTareas {
    constructor() {
        this.tareas = this.cargarTareas();
    }
    
    agregarTarea(texto, prioridad = 'media') {
        const tarea = {
            id: Date.now(),
            texto,
            prioridad,
            completada: false,
            fechaCreacion: new Date().toISOString()
        };
        
        this.tareas.push(tarea);
        this.guardarTareas();
        return tarea;
    }
    
    completarTarea(id) {
        const tarea = this.tareas.find(t => t.id === id);
        if (tarea) {
            tarea.completada = true;
            this.guardarTareas();
        }
    }
    
    eliminarTarea(id) {
        this.tareas = this.tareas.filter(t => t.id !== id);
        this.guardarTareas();
    }
    
    obtenerTareas(filtro = 'todas') {
        switch (filtro) {
            case 'completadas':
                return this.tareas.filter(t => t.completada);
            case 'pendientes':
                return this.tareas.filter(t => !t.completada);
            default:
                return this.tareas;
        }
    }
    
    guardarTareas() {
        localStorage.setItem('tareas', JSON.stringify(this.tareas));
    }
    
    cargarTareas() {
        const tareas = localStorage.getItem('tareas');
        return tareas ? JSON.parse(tareas) : [];
    }
}

// Uso del gestor de tareas
const gestor = new GestorTareas();
gestor.agregarTarea('Aprender JavaScript', 'alta');
gestor.agregarTarea('Hacer ejercicio', 'media');
console.log(gestor.obtenerTareas());

if/else - Condicionales

¿Qué es if/else?

Es la estructura de control más básica que permite que tu código tome decisiones. Evalúa una condición y ejecuta diferentes bloques de código según el resultado.

Sintaxis:

if (condición) {
    // Código si la condición es verdadera
} else if (otra_condición) {
    // Código si la segunda condición es verdadera
} else {
    // Código si ninguna condición es verdadera
}

Ejemplo Práctico:

let edad = 18;

if (edad >= 18) {
    console.log("Eres mayor de edad");
} else if (edad >= 13) {
    console.log("Eres adolescente");
} else {
    console.log("Eres menor de edad");
}

Puntos Clave:

  • La condición debe evaluar a true o false
  • Puedes tener múltiples else if
  • El else es opcional
  • Usa {} para bloques de múltiples líneas

switch/case - Múltiples Casos

¿Qué es switch y case?

switch es una estructura que evalúa una expresión y ejecuta diferentes bloques de código según el valor. case define cada posible valor y qué hacer en cada caso.

Sintaxis:

switch (expresión) {
    case valor1:
        // Código para valor1
        break;
    case valor2:
        // Código para valor2
        break;
    default:
        // Código si no coincide con ningún caso
        break;
}

¿Qué hace cada parte?

  • switch (expresión): Evalúa la expresión una sola vez
  • case valor: Compara el valor con la expresión
  • break: Sale del switch (¡MUY IMPORTANTE!)
  • default: Se ejecuta si ningún case coincide

Ejemplo Práctico:

let dia = "lunes";

switch (dia) {
    case "lunes":
        console.log("Inicio de semana");
        break;
    case "viernes":
        console.log("¡Fin de semana!");
        break;
    case "sábado":
    case "domingo":
        console.log("Día de descanso");
        break;
    default:
        console.log("Día laboral");
}

⚠️ ¡Cuidado con el break!

Sin break, el código "cae" al siguiente case (fall-through). Esto puede ser útil o causar errores.

for - Bucle con Contador

¿Qué es el bucle for?

Es un bucle que se repite un número específico de veces usando un contador. Es perfecto cuando sabes exactamente cuántas veces quieres que se ejecute el código.

Sintaxis:

for (inicialización; condición; actualización) {
    // Código que se repite
}

¿Qué hace cada parte?

  • inicialización: Se ejecuta UNA VEZ al inicio (ej: let i = 0)
  • condición: Se evalúa ANTES de cada iteración (ej: i < 5)
  • actualización: Se ejecuta DESPUÉS de cada iteración (ej: i++)

Ejemplo Paso a Paso:

for (let i = 0; i < 3; i++) {
    console.log(`Iteración ${i}`);
}

// Resultado:
// Iteración 0
// Iteración 1
// Iteración 2

Proceso paso a paso:

  1. Inicio: i = 0
  2. ¿i < 3? Sí (0 < 3) → Ejecuta código → i++ → i = 1
  3. ¿i < 3? Sí (1 < 3) → Ejecuta código → i++ → i = 2
  4. ¿i < 3? Sí (2 < 3) → Ejecuta código → i++ → i = 3
  5. ¿i < 3? No (3 < 3) → Termina el bucle

Usos Comunes:

  • Iterar arrays con índices: for (let i = 0; i < array.length; i++)
  • Repetir código N veces
  • Generar secuencias numéricas

for...of - Iterar VALORES

¿Qué es for...of?

Es un bucle que itera sobre los VALORES de elementos iterables (arrays, strings, etc.). Es más simple que el for tradicional cuando solo necesitas los valores, no los índices.

Sintaxis:

for (let elemento of iterable) {
    // Código que usa el elemento
}

Ejemplo con Array:

let frutas = ["manzana", "banana", "naranja"];

for (let fruta of frutas) {
    console.log(fruta);
}

// Resultado:
// manzana
// banana
// naranja

Comparación: for vs for...of

for tradicional (con índices):

for (let i = 0; i < frutas.length; i++) {
    console.log(`${i}: ${frutas[i]}`);
}
// Resultado:
// 0: manzana
// 1: banana
// 2: naranja

for...of (solo valores):

for (let fruta of frutas) {
    console.log(fruta);
}
// Resultado:
// manzana
// banana
// naranja

¿Cuándo usar for...of?

  • ✅ Cuando solo necesitas los valores
  • ✅ Código más limpio y legible
  • ✅ Menos propenso a errores
  • ❌ NO cuando necesitas el índice
  • ❌ NO para modificar el array original

for...in - Iterar PROPIEDADES

¿Qué es for...in?

Es un bucle que itera sobre las PROPIEDADES (claves) de un objeto. Te da acceso a los nombres de las propiedades, no a sus valores.

Sintaxis:

for (let propiedad in objeto) {
    // Código que usa la propiedad
}

Ejemplo con Objeto:

let persona = {
    nombre: "Juan",
    edad: 25,
    ciudad: "Madrid"
};

for (let propiedad in persona) {
    console.log(`${propiedad}: ${persona[propiedad]}`);
}

// Resultado:
// nombre: Juan
// edad: 25
// ciudad: Madrid

¿Cómo acceder a los valores?

Usa la notación de corchetes: objeto[propiedad]

for (let prop in persona) {
    console.log(prop);                    // nombre, edad, ciudad
    console.log(persona[prop]);          // Juan, 25, Madrid
    console.log(persona.prop);           // undefined (¡ERROR!)
}

Diferencia clave: for...of vs for...in

for...of (valores):

let array = [10, 20, 30];
for (let valor of array) {
    console.log(valor);  // 10, 20, 30
}

for...in (propiedades):

let array = [10, 20, 30];
for (let indice in array) {
    console.log(indice);  // 0, 1, 2
    console.log(array[indice]); // 10, 20, 30
}

⚠️ Advertencias importantes:

  • for...in también itera propiedades heredadas
  • El orden no está garantizado en objetos
  • Para arrays, usa for...of o for tradicional

while - Mientras se cumple condición

¿Qué es el bucle while?

Es un bucle que se repite mientras una condición sea verdadera. No sabes cuántas veces se ejecutará, solo sabes cuándo debe parar.

Sintaxis:

while (condición) {
    // Código que se repite
    // ¡IMPORTANTE! Modificar la condición aquí
}

Ejemplo Básico:

let contador = 0;

while (contador < 3) {
    console.log(`Contador: ${contador}`);
    contador++;  // ¡IMPORTANTE! Sin esto sería un bucle infinito
}

// Resultado:
// Contador: 0
// Contador: 1
// Contador: 2

Proceso paso a paso:

  1. ¿contador < 3? Sí (0 < 3) → Ejecuta → contador = 1
  2. ¿contador < 3? Sí (1 < 3) → Ejecuta → contador = 2
  3. ¿contador < 3? Sí (2 < 3) → Ejecuta → contador = 3
  4. ¿contador < 3? No (3 < 3) → Termina

⚠️ ¡Cuidado con los bucles infinitos!

// ❌ MAL - Bucle infinito
let i = 0;
while (i < 5) {
    console.log("Hola");
    // ¡Falta i++!
}

// ✅ BIEN - Condición que cambia
let i = 0;
while (i < 5) {
    console.log("Hola");
    i++;  // La condición cambia
}

¿Cuándo usar while?

  • ✅ Cuando no sabes cuántas iteraciones necesitas
  • ✅ Validación de entrada del usuario
  • ✅ Procesar datos hasta encontrar un valor específico
  • ❌ NO cuando sabes exactamente cuántas veces iterar (usa for)

Operador Ternario (? :)

¿Qué es el operador ternario?

Es una forma abreviada de escribir un if/else simple en una sola línea. Es perfecto para asignaciones condicionales.

Sintaxis:

condición ? valor_si_verdadero : valor_si_falso

Comparación: if/else vs Ternario

if/else tradicional:

let edad = 18;
let mensaje;

if (edad >= 18) {
    mensaje = "Mayor de edad";
} else {
    mensaje = "Menor de edad";
}

Operador ternario:

let edad = 18;
let mensaje = edad >= 18 ? "Mayor de edad" : "Menor de edad";

Ejemplos Prácticos:

// Asignación condicional
let precio = 100;
let descuento = precio > 50 ? 10 : 0;

// En template strings
let nombre = "Juan";
let saludo = `Hola ${nombre}, ${nombre.length > 3 ? "bienvenido" : "hola"}`;

// Retorno de función
function esPar(numero) {
    return numero % 2 === 0 ? "par" : "impar";
}

⚠️ No abuses del ternario:

// ❌ MAL - Muy difícil de leer
let resultado = a > b ? c > d ? "A" : "B" : c > d ? "C" : "D";

// ✅ BIEN - Usa if/else para casos complejos
let resultado;
if (a > b) {
    resultado = c > d ? "A" : "B";
} else {
    resultado = c > d ? "C" : "D";
}

¿Cuándo usar ternario?

  • ✅ Asignaciones simples (una condición)
  • ✅ Valores por defecto
  • ✅ Template strings
  • ❌ NO para lógica compleja
  • ❌ NO para múltiples condiciones
