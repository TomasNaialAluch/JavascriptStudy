Let, Const y Block Scope

if (true) {
let variableLet = "Solo en este bloque";
const variableConst = "Constante en bloque";
var variableVar = "Accesible fuera del bloque";
}
console.log(variableVar); // ✅ Funciona
console.log(variableLet); // ❌ Error
let variableLet
const variableConst
var variableVar
variableLet (no existe)
variableVar (accesible)
Block Scope: let y const solo existen dentro del bloque

Explicación del Scope

El scope (alcance) determina dónde se puede acceder a una variable:

  • var: Function scope - accesible en toda la función
  • let/const: Block scope - solo accesible dentro del bloque {}
  • Bloque: Cualquier código entre llaves {}

Arrow Functions - Sintaxis Moderna

// Función tradicional
function sumar(a, b) {
return a + b;
}
// Arrow function
const sumar = (a, b) => a + b;
function sumar(a, b) { return a + b; }
const sumar = (a, b) => a + b
✅ Más corta
✅ Sin this propio
✅ Return implícito

Explicación de Arrow Functions

Las arrow functions son una sintaxis más corta para escribir funciones:

  • Sintaxis: (parámetros) => expresión
  • Return implícito: Si es una expresión, no necesita return
  • Sin this: Hereda el this del contexto padre
  • Ideal para: Callbacks, funciones simples, métodos de array

Template Literals - Strings Inteligentes

let nombre = "Ana";
let edad = 25;
// Concatenación tradicional
let mensaje1 = "Hola " + nombre + ", tienes " + edad + " años";
// Template literal
let mensaje2 = `Hola ${nombre}, tienes ${edad} años`;
nombre = "Ana"
edad = 25
"Hola " + nombre + ", tienes " + edad + " años"
`Hola ${nombre}, tienes ${edad} años`
Resultado: "Hola Ana, tienes 25 años"

Explicación de Template Literals

Los template literals permiten crear strings de forma más elegante:

  • Backticks: Usa ` en lugar de comillas
  • Interpolación: ${variable} inserta el valor
  • Multilínea: Pueden contener saltos de línea
  • Expresiones: ${2 + 3} evalúa la expresión
